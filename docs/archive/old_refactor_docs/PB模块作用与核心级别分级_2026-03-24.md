# PB 模块作用与核心级别分级（重构前基线）

## 1. 目的

- 在 PB→Python 重构启动前，先明确各 `.pbl` 目录在 ITSM 系统中的业务作用与核心级别。
- 以“核心优先、依赖优先、风险可控”为原则，避免跨模块返工与重复改造。

## 2. 分级规则

- C0（核心）：直接影响登录鉴权、主框架导航、ITSM 主工单流转、库存交易主链路。
- C1（重要）：支撑核心链路的业务域模块，或高频但可分批切换的模块。
- C2（外围）：低频、报表/工具/导出类或可后置迁移模块。

> 说明：本分级用于“迁移顺序决策”，不等同于业务价值高低；后续可在每个模块调研后微调。

## 3. 模块作用与优先级建议

## 4. 建议分批迁移（Wave）

- Wave 1（P0）：`app_main.pbl`、`app_system.pbl`、`base_srv.pbl`、`itsm.pbl`、`itsm02.pbl`、`base_cust.pbl`
- Wave 2（P1）：`sale.pbl`、`purchase.pbl`、`wh.pbl`、`trans.pbl`、`qc.pbl`、`price.pbl`、`base_objects.pbl`、`base_attrib.pbl`、`new_deposit.pbl`、`wh_syc.pbl`
- Wave 3（P2）：`report.pbl`、`kq.pbl`、`appraisal.pbl`、`lsgldata.pbl`、`dw2xls.pbl`、`app_d.pbl`、`app_dddw.pbl`、`app_objects.pbl`、`base_dwsrv.pbl`

## 5. 执行要求

- 每个 Wave 内仍按“模块最小可交付”推进，禁止跨模块混迁。
- 每个模块进入开发前，必须补齐：
  - `docs/PB_TO_PYTHON_MODULE_MAPPING.csv`
  - `docs/PB_TO_PYTHON_SQL_MAPPING.csv`
  - `docs/PB_TO_PYTHON_DOD_CHECKLIST.md` 中对应验收项
- 如分级与业务现状冲突，先在 `docs` 补充差异说明并更新本文件后再调整排期。
