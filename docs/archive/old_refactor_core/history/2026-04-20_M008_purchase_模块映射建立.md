## 数据库表结构分析

### 1. 采购计划（TPC01_PCPLAN 主表 + TPC02_PCPLANDT 明细表 + TPC03_PCPLANSTATUS 状态表）

**TPC01_PCPLAN（采购计划主表）**
- PCPLANID (CHAR(8)): 采购计划ID（PK）
- SLBILLID (CHAR(8)): 销售单据ID
- PCTYP (CHAR(1)): 采购类型
- PTIMES (NUMBER): 采购次数
- OPERCD (CHAR(6)): 操作员代码
- AUDITFLG (CHAR(1)): 审核标志
- TYPE (CHAR(1)): 类型

**TPC02_PCPLANDT（采购计划明细表）**
- PCPLANID (CHAR(8)): 采购计划ID（PK）
- LINENO (NUMBER): 行号（PK）
- ITEMCD (CHAR(6)): 物料代码
- PLANQTY (NUMBER(12,0)): 计划数量
- UPPERLIMIT (NUMBER(12,0)): 上限
- AUDITQTY (NUMBER(12,0)): 审核数量

**TPC03_PCPLANSTATUS（采购计划状态表）**
- ITEMCD (CHAR(6)): 物料代码（PK）
- RGSTQTY (NUMBER(12,0)): 登记数量
- AUDITQTY (NUMBER(12,0)): 审核数量
- PCQTY (NUMBER(12,0)): 采购数量
- OPERCD (CHAR(6)): 操作员代码
- MEMO (VARCHAR2(255)): 备注
- UPDDATE (DATE): 更新日期
- REFBILLID (CHAR(8)): 参考单据ID

### 2. 采购登记（TPC12_REGISTER 主表 + TPC13_REGISTERDT 明细表）

**TPC12_REGISTER（采购登记主表）**
- RGSTBILLID (CHAR(8)): 登记单据ID（PK）
- SUPPLIERCD (CHAR(8)): 供应商代码
- PCREP (CHAR(6)): 采购代表
- PTIMES (NUMBER): 采购次数
- OPERCD (CHAR(6)): 操作员代码
- RGSTDATE (DATE): 登记日期
- RGSTAMT (NUMBER(12,2)): 登记金额

**TPC13_REGISTERDT（采购登记明细表）**
- RGSTBILLID (CHAR(8)): 登记单据ID（PK）
- LINENO (NUMBER): 行号（PK）
- ITEMCD (CHAR(6)): 物料代码

### 3. 采购单据（TPC14_PCBILL）

**TPC14_PCBILL（采购单据表）**
- PCBILLID (CHAR(8)): 采购单据ID（PK）
- PCTYP (CHAR(2)): 采购类型
- CUSTCD (CHAR(8)): 客户代码
- REFBILLID (CHAR(8)): 参考单据ID
- PCDATE (DATE): 采购日期
- PCAMT (NUMBER(16,4)): 采购金额
- WHCD (CHAR(2)): 仓库代码
- INVOICEFLG (CHAR(1)): 发票标志

### 4. 入库单据（TPC16_RPCBILL 主表 + TPC17_RPCBILLDT 明细表）

**TPC16_RPCBILL（入库单据表）**
- PCBILLID (CHAR(8)): 采购单据ID（PK）
- CUSTCD (CHAR(8)): 客户代码
- PCDATE (DATE): 采购日期
- PCAMT (NUMBER(12,0)): 采购金额
- WHCD (CHAR(2)): 仓库代码
- INVOICEFLG (CHAR(2)): 发票标志

**TPC17_RPCBILLDT（入库单据明细表）**
- PCBILLID (CHAR(8)): 采购单据ID（PK）
- LINENO (NUMBER): 行号（PK）
- ITEMTYP (CHAR(2)): 物料类型
- EID (CHAR(13)): 设备ID
- SEID (VARCHAR2(30)): 序列设备ID
- RPCQTY (NUMBER(12,0)): 入库数量
- INVOICEQTY (NUMBER(12,0)): 发票数量
- UNITS (VARCHAR2(4)): 单位

### 5. 供应商评价（TPC20_SUPPAPPRAISAL 主表 + TPC21_SUPPAPPRAISALDT 明细表）

**TPC20_SUPPAPPRAISAL（供应商评价主表）**
- SUPPLIERID (CHAR(8)): 供应商ID（PK）
- APPDATE (DATE): 评价日期
- APPREPCD (CHAR(6)): 评价代表代码
- APPMEMO (VARCHAR2(255)): 评价备注
- APPFLG (CHAR(1)): 评价标志

**TPC21_SUPPAPPRAISALDT（供应商评价明细表）**
- SUPPLIERID (CHAR(8)): 供应商ID（PK）
- LINENO (NUMBER): 行号（PK）
- SUPPLIERCD (CHAR(8)): 供应商代码
- APPCODE (CHAR(2)): 评价代码
- APPSCORE (NUMBER): 评价分数
- APPFLG (CHAR(1)): 评价标志

## 业务流程推断

### 采购计划流程
1. 创建采购计划（TPC01_PCPLAN + TPC02_PCPLANDT）
2. 采购计划审核（AUDITFLG 状态变更）
3. 采购计划状态跟踪（TPC03_PCPLANSTATUS）

### 采购登记流程
1. 创建采购登记（TPC12_REGISTER + TPC13_REGISTERDT）
2. 采购登记审批（u_pc_register_examine）
3. 生成采购单据（TPC14_PCBILL）

### 入库流程
1. 生成入库单据（TPC16_RPCBILL + TPC17_RPCBILLDT）
2. 入库数量确认
3. 发票确认（INVOICEFLG）

### 供应商评价流程
1. 创建供应商评价（TPC20_SUPPAPPRAISAL + TPC21_SUPPAPPRAISALDT）
2. 评价分数录入
3. 评价状态管理
