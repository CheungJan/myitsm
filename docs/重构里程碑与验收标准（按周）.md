# 重构里程碑与验收标准（按周）

> 文档版本：v0.1  
> 适用项目：myitsm（PB -> Python）  
> 重构策略：共存迁移，分域替换，双轨运行

---

## 1. 总体目标

1. 不中断现网业务；
2. 先替换高价值高痛点模块（登录权限、ITSM核心）；
3. 每周可演示、每阶段可回滚。

---

## 2. 周度计划（建议12周）

## 3. 全程验收门槛

1. 功能一致性：核心用例通过率 >= 95%；
2. 数据一致性：关键单据字段一致率 >= 99.5%；
3. 性能：关键接口P95满足业务目标；
4. 质量：测试覆盖率 >= 85%（核心 >= 90%）；
5. 安全：无高危漏洞、无明文密钥。

---

## 4. 风险与回滚

### 4.1 高风险点
- 状态码含义历史不一致；
- 存储过程黑盒依赖；
- 菜单权限与对象映射偏差。

### 4.2 回滚原则
1. 按模块开关回退，不做全局硬切；
2. 保留 PB 同步入口，确保业务可继续；
3. 关键表变更均配套可逆脚本。
