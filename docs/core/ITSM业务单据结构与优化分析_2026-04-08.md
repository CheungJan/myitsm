## 3. 公用附表详解

### 3.1 TIT23_MAINTENANCE_D2D（上门服务记录）

**跨单据复用**：MD/MO/MR/BY/BG 均使用此表记录上门服务

**源码验证**（`d_divechange_report.srd` 第81-83行）：
```sql
LEFT JOIN (SELECT max(ARRIVE_TIME) ARRIVE_TIME, MAINTENANCE_ID 
           FROM TIT23_MAINTENANCE_D2D 
           GROUP BY MAINTENANCE_ID) T 
  ON M.DEVICE_CHANGE_ID = T.MAINTENANCE_ID
```

**核心字段**：
- MAINTENANCE_ID：关联各单据主键
- D2D_TYPE：类型（1到店/2离店/3催单/4记录）
- D2D_ENGINEER：上门工程师
- ARRIVE_TIME：到达时间
- LEAVE_TIME：离店时间
- D2D_DESCRIPITON：跟踪描述

## 4. 设备变更单（TIT16）专题分析

### 4.1 表结构

**主表**：TIT16_DEVICE_CHANGE

| 字段名 | 类型 | 说明 |
|--------|------|------|
| DEVICE_CHANGE_ID | VARCHAR2(8) | **主键** - 变更单ID |
| STORE_ID | VARCHAR2(8) | **原门店ID**（关联TMM22） |
| REQUSET_PAPER_ID | VARCHAR2(8) | 变更请求单ID |
| **CHANGE_TYPE** | VARCHAR2(8) | **变更类型** |
| DEVICE_ID | CHAR(13) | 整机ID |
| NEW_CONTACTOR | VARCHAR2(10) | 变更后联系人 |
| NEW_TEL | VARCHAR2(60) | 变更后电话 |
| NEW_ADDRESS | VARCHAR2(200) | 变更后地址 |
| **NEW_STORE_CARD** | VARCHAR2(20) | **变更后门店磁卡号** |
| **NEW_STORE_ID** | VARCHAR2(8) | **变更后门店ID**（跨门店时使用） |
| IS_STORE_INSIDE_CHANGE | CHAR(1) | 是否店内移机（Y/N） |
| CURRENT_STATUS | CHAR(1) | 当前状态 |

#### 变更类型（CHANGE_TYPE）解码

**源码出处**（`d_itsm_khhz.srd` 第17-44行）：
```pb
DECODE(CHANGE_TYPE,
  'CK', '改磁卡号',
  'BQ', '信息变更', 
  'BG', '设备变更',
  ...
)
```

| 类型码 | 含义 | 业务场景 |
|--------|------|---------|
| **CK** | 改磁卡号 | 同门店，仅磁卡号变更 |
| **BQ** | 信息变更 | 联系人/电话/地址变更 |
| **BG** | 设备变更 | 跨门店设备迁移 |

#### 公用附表

| 表名 | 关联字段 | 用途 |
|------|---------|------|
| TIT23_MAINTENANCE_D2D | MAINTENANCE_ID = DEVICE_CHANGE_ID | 上门服务记录 |

**⚠️ 无独立子表**：设备变更单是"动作记录"，非"设备清单"

---

### 4.2 老系统问题分析

#### 问题现象

当 **CHANGE_TYPE = 'CK'（改磁卡号）** 时：
- 门店未变（STORE_ID 不变）
- 仅磁卡号从旧值变为新值（NEW_STORE_CARD）
- **老系统处理**：直接删除 TMM22_CUSTOMERS 中旧磁卡号记录

#### 隐患

1. **历史数据丢失**：无法追溯"某磁卡号曾经属于哪家门店"
2. **审计追溯困难**：无法查看门店磁卡号变更历史
3. **报表数据缺失**：按旧磁卡号查询时记录不存在

---

### 4.3 优化方案

#### 方案1：新增历史记录表（推荐）

**新增表**：`TMM22_CUSTOMERS_HISTORY`

```sql
CREATE TABLE TMM22_CUSTOMERS_HISTORY (
    HISTORY_ID          NUMBER PRIMARY KEY,      -- 历史记录ID
    CUSTCD              CHAR(8),                 -- 客户代码（门店ID）
    CUSTCARD            VARCHAR2(20),            -- 磁卡号（当前值）
    CUSTNM              VARCHAR2(60),            -- 客户名称
    STORE_ID            VARCHAR2(8),             -- 所属门店
    
    -- 变更追踪
    CHANGE_TYPE         CHAR(2),                 -- CK=磁卡号, BQ=信息, BG=设备
    CHANGE_REASON       VARCHAR2(200),           -- 变更原因
    OLD_CUSTCARD        VARCHAR2(20),            -- 原磁卡号
    NEW_CUSTCARD        VARCHAR2(20),            -- 新磁卡号
    CHANGE_TIME         DATE,                    -- 变更时间
    DEVICE_CHANGE_ID    VARCHAR2(8),             -- 关联变更单ID
    OPER_CD             CHAR(6),                 -- 操作人
    
    USEFLG              CHAR(1) DEFAULT '1',     -- 有效标志
    CREATE_TIME         DATE DEFAULT SYSDATE,
    UPDATE_TIME         DATE DEFAULT SYSDATE
);

-- 索引
CREATE INDEX IDX_CUST_HIST_CARD ON TMM22_CUSTOMERS_HISTORY(CUSTCARD);
CREATE INDEX IDX_CUST_HIST_STORE ON TMM22_CUSTOMERS_HISTORY(STORE_ID);
CREATE INDEX IDX_CUST_HIST_CHANGE ON TMM22_CUSTOMERS_HISTORY(CHANGE_TYPE, CHANGE_TIME);
CREATE INDEX IDX_CUST_HIST_DEVICE ON TMM22_CUSTOMERS_HISTORY(DEVICE_CHANGE_ID);
```

#### 优化后流程

```
┌─────────────────────────────────────────────────────────────┐
│  设备变更单创建（CHANGE_TYPE='CK'）                          │
└──────────────────┬────────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  1. 插入 TMM22_CUSTOMERS_HISTORY（保存旧磁卡号信息）          │
│     OLD_CUSTCARD = 当前CUSTCARD                              │
│     NEW_CUSTCARD = 变更单.NEW_STORE_CARD                      │
└──────────────────┬────────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 更新 TMM22_CUSTOMERS                                     │
│     CUSTCARD = 变更单.NEW_STORE_CARD                          │
│     （保留门店主体信息）                                      │
└──────────────────┬────────────────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  3. 更新设备变更单状态 → COMPLETED                           │
└─────────────────────────────────────────────────────────────┘
```

#### 查询支持

**场景1：查询某磁卡号当前归属**
```sql
SELECT * FROM TMM22_CUSTOMERS WHERE CUSTCARD = :card_no;
```

**场景2：查询磁卡号变更历史**
```sql
SELECT * FROM TMM22_CUSTOMERS_HISTORY 
WHERE OLD_CUSTCARD = :card_no OR NEW_CUSTCARD = :card_no
ORDER BY CHANGE_TIME DESC;
```

**场景3：查询门店磁卡号变更时间线**
```sql
SELECT * FROM TMM22_CUSTOMERS_HISTORY 
WHERE STORE_ID = :store_id AND CHANGE_TYPE = 'CK'
ORDER BY CHANGE_TIME DESC;
```
