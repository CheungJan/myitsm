# 多点登录功能故障排查手册

## 功能概述

多点登录控制基于 `tmc71_sysparm.allowmultilogon` 参数实现，通过 `tmc41_acclog` 表管理活跃 token 状态。

-- **允许多点登录（allowmultilogon=1）**：同一账号可在多台设备同时登录，各设备独立持有 token
-- **禁止多点登录（allowmultilogon=0）**：同一时刻只允许一台设备在线，新登录设备会被拒绝

---

## 数据结构

### tmc41_acclog 表 token 记录

```sql
SELECT id, user_cd, action, LEFT(detail, 30) AS token_prefix, startdate, created_at
FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN'
ORDER BY created_at DESC;
```

**字段说明：**
- `action='ACTIVE_TOKEN'`：标识活跃 token 记录
- `detail`：存储 JWT token 完整字符串
- `startdate`：存储 token 过期时间（UTC）
- `created_at`：记录创建时间

**正常状态：**
- 允许多点登录：同一用户可能有 1-N 条记录（对应 N 台设备）
- 禁止多点登录：同一用户最多 1 条记录

## 常见故障排查

### 故障 1：设置禁止多点登录后，所有设备都无法登录

**现象：**
```
allowmultilogon = 0
Mac 登录成功
iPhone 登录 → 提示 "该账号已在其他设备登录，管理员禁止多点登录"
Mac 再次登录 → 也提示相同错误
```

**原因：**
早期实现检查 8 小时内是否有 LOGIN 日志且无 LOGOUT，两台设备都登录后互相认为对方在线，导致互相拒绝。

**排查步骤：**
```sql
-- 查看当前活跃 token
SELECT user_cd, action, startdate, created_at
FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN';
```

**解决方案：**
当前实现已改为基于 JWT token 精确管理，不会出现此问题。如遇类似情况：
1. 检查 `allowmultilogon` 参数值
2. 查询 `ACTIVE_TOKEN` 记录确认是否有残留
3. 如有残留，手动清理：
```sql
DELETE FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN'
  AND user_cd = 'admin';
```

### 故障 2：退出后数据库仍有 ACTIVE_TOKEN 记录

**现象：**
```
Mac 登录 → 1 条 ACTIVE_TOKEN
iPhone 登录 → 2 条 ACTIVE_TOKEN
Mac 退出 → 仍为 2 条 ACTIVE_TOKEN
```

**原因排查：**

**原因 A：前端退出请求未成功发送**
- 检查浏览器 Network 面板，确认是否有 `POST /api/v1/logout` 请求
- 检查请求状态码是否为 200
- 检查请求是否带 `Authorization: Bearer <token>` header

**原因 B：后端未正确删除 token**
- 检查后端日志是否有 `/logout` 请求记录
- 检查 `auth_repository.revoke_token()` 是否被调用
- 检查 token 是否匹配（detail 字段与 header 中的 token 一致）

**解决方案：**

1. **前端修复**（已实现）：
```typescript
// auth.ts
async function logout() {
    await logoutApi()  // 确保请求完成
    token.value = ''
    // ...
}
```

2. **后端修复**（已实现）：
```python
# auth.py
def logout():
    token = auth_header[7:].strip()
    AuthService.logout(user_cd, token)  # 精确删除当前 token
```

3. **手动清理**：
```sql
DELETE FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN'
  AND detail = '<具体 token 字符串>';
```

### 故障 3：允许多点登录时，后登录设备踢掉前一台设备

**现象：**
```
allowmultilogon = 1
Mac 登录 → token_A
iPhone 登录 → token_B，但 token_A 被删除
Mac 刷新页面 → 提示 401
```

**原因：**
早期实现每次登录都会 `revoke_user_tokens(user_cd)` 删除该用户所有 token，导致允许多点登录模式下也变成单点登录。

**排查步骤：**
```sql
-- 检查是否只有 1 条记录
SELECT COUNT(*) FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN' AND user_cd = 'admin';
```

**解决方案：**
当前实现已修复，只在禁止多点登录时才删除旧 token：
```python
if not allow_multi:
    _repo.revoke_user_tokens(user_id)
_repo.register_token(user_id, token, expires_at)
```

### 故障 4：退出后旧 token 仍能访问 API

**现象：**
```
Mac 登录 → token_A
Mac 退出 → token_A 已删除
Mac 复制 token_A 手动调用 API → 仍然成功
```

**原因：**
JWT 本身未过期，数据库删除 token 后，API 接口没有检查 token 是否仍在活跃表中。

**解决方案：**
当前实现已在 `get_session()` 中加双重验证：
```python
def get_session(token: str):
    payload = validate_token(token)  # JWT 验证
    if not is_token_active(token):  # 数据库活跃性验证
        return None
```

---

### 故障 5：数据库累积大量过期 ACTIVE_TOKEN 记录

**现象：**
```sql
SELECT COUNT(*) FROM tmc41_acclog WHERE action = 'ACTIVE_TOKEN';
-- 结果：几千条记录
```

**原因：**
token 过期后没有被自动清理，导致数据库累积无效记录。

**解决方案：**
当前实现已在以下时机自动清理：
1. 每次登录前：`clean_expired_tokens()`
2. 每次登出时：`revoke_token()` 调用 `clean_expired_tokens()`

**手动清理（如需）：**
```sql
DELETE FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN'
  AND startdate <= NOW();
```

## 日常运维 SQL

### 查看当前在线用户

```sql
SELECT DISTINCT user_cd
FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN'
  AND startdate > NOW();
```

### 查看某用户的所有活跃 token

```sql
SELECT id, LEFT(detail, 30) AS token_prefix, startdate, created_at
FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN'
  AND user_cd = 'admin'
ORDER BY created_at DESC;
```

### 强制踢出某用户所有设备

```sql
DELETE FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN'
  AND user_cd = 'admin';
```

### 强制踢出指定 token

```sql
DELETE FROM tmc41_acclog
WHERE action = 'ACTIVE_TOKEN'
  AND detail = '<完整 JWT token 字符串>';
```

### 查看系统参数当前设置

```sql
SELECT allowmultilogon FROM tmc71_sysparm WHERE parm_cd = 'SYSPARM';
```

### 修改多点登录策略

```sql
UPDATE tmc71_sysparm
SET allowmultilogon = '0'  -- 0=禁止, 1=允许
WHERE parm_cd = 'SYSPARM';
```

## 前端调试技巧

### 检查退出请求

1. 打开浏览器开发者工具 → Network 面板
2. 点击退出按钮
3. 查找 `POST /api/v1/logout` 请求
4. 检查：
   - Request Headers 是否包含 `Authorization: Bearer ...`
   - Status Code 是否为 200
   - Response 是否为 `{ code: 200, message: "已登出" }`

### 检查登录响应

1. Network 面板查找 `POST /api/v1/login`
2. 检查 Response 中的 `token` 字段
3. 复制 token 用于数据库查询

### 本地存储检查

```javascript
// 浏览器 Console 执行
console.log(localStorage.getItem('token'))
```
