# 多点登录功能技术实现方案

## 1. 需求背景

### 业务需求
系统需要支持两种多点登录策略：
- **允许多点登录**：同一账号可在多台设备同时登录（如 PC、手机、平板）
- **禁止多点登录**：同一时刻只允许一台设备在线，新登录设备会被拒绝

### 技术约束
- 使用 JWT（JSON Web Token）进行无状态认证
- 不引入新的数据库表，复用现有 `tmc41_acclog` 表
- 保持与现有认证架构兼容
- 支持前端 Vue 3 + Element Plus

## 3. 数据模型设计

### 3.1 复用 tmc41_acclog 表

**原表结构：**
```sql
CREATE TABLE tmc41_acclog (
    id SERIAL PRIMARY KEY,
    user_cd VARCHAR(20),
    action VARCHAR(50),
    detail TEXT,
    ip_addr VARCHAR(50),
    startdate DATETIME,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**字段复用策略：**

| 字段 | 原用途 | token 管理用途 |
|------|--------|----------------|
| `action` | 操作类型（LOGIN/LOGOUT） | `ACTIVE_TOKEN` 标识活跃 token |
| `detail` | 操作明细 | 存储 JWT token 完整字符串 |
| `startdate` | 开始日期（Oracle 原字段） | 存储 token 过期时间（UTC） |
| `user_cd` | 用户编码 | 关联用户 |

**优势：**
- 无需新建表，最小化数据库变更
- `detail` 字段为 TEXT 类型，可存储完整 JWT
- `startdate` 字段恰好可用于存储过期时间

## 4. 核心流程实现

### 4.1 登录流程

```
┌─────────────────────────────────────────────────────────────┐
│  POST /api/v1/login                                         │
│  { "user_id": "admin", "password": "xxx" }                  │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│  AuthService.login()                                        │
│  1. 验证用户名密码                                          │
│  2. clean_expired_tokens()  ← 清理过期 token                │
│  3. 读取 allowmultilogon 参数                                │
│  4. if allowmultilogon == '0':                             │
│        if has_active_token(user_id):                        │
│            return {"error": "该账号已在其他设备登录"}        │
│  5. 生成 JWT token                                          │
│  6. if allowmultilogon == '0':                             │
│        revoke_user_tokens(user_id)  ← 删除旧 token          │
│  7. register_token(user_id, token, expires_at)              │
│  8. 写 LOGIN 日志                                           │
│  9. 返回 token                                              │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│  数据库操作                                                 │
│  DELETE FROM tmc41_acclog WHERE action='ACTIVE_TOKEN'      │
│        AND startdate <= NOW();  ← 清理过期                  │
│  INSERT INTO tmc41_acclog (action='ACTIVE_TOKEN',           │
│        detail=token, startdate=expires_at)                   │
└─────────────────────────────────────────────────────────────┘
```

**关键代码：**
```python
# app/services/auth_service.py
@staticmethod
def login(user_id: str, password: str) -> dict[str, Any] | None:
    # 1. 验证密码
    user = _repo.get_user(user_id)
    if not check_password_hash(user.password, password):
        return None
    
    # 2. 清理过期 token
    _repo.clean_expired_tokens()
    
    # 3. 多点登录检查
    sp = db.session.get(SysParm, "SYSPARM")
    allow_multi = (sp is None) or (sp.allowmultilogon != "0")
    if not allow_multi:
        if _repo.has_active_token(user_id):
            return {"error": "该账号已在其他设备登录，管理员禁止多点登录"}
    
    # 4. 生成 token
    token = _generate_token(user_id)
    exp_seconds = int(current_app.config.get("JWT_EXPIRATION_SECONDS", 28800))
    expires_at = datetime.now(timezone.utc).replace(tzinfo=None) + timedelta(seconds=exp_seconds)
    
    # 5. Token 注册策略
    if not allow_multi:
        _repo.revoke_user_tokens(user_id)  # 禁止：删旧 token
    _repo.register_token(user_id, token, expires_at)  # 写入新 token
    
    # 6. 写日志
    _repo.add_access_log(user_cd=user_id, action="LOGIN", detail="用户登录")
    db.session.commit()
    
    return {"token": token, "user_code": user.user_cd, ...}
```

└────────────────────┬────────────────────────────────────────┘

│  数据库操作                                                 │
│  DELETE FROM tmc41_acclog WHERE action='ACTIVE_TOKEN'      │
│        AND detail = '<当前 token>';  ← 精确删除              │
│  DELETE FROM tmc41_acclog WHERE action='ACTIVE_TOKEN'      │
│        AND startdate <= NOW();  ← 清理过期                  │
└─────────────────────────────────────────────────────────────┘

**关键代码：**
```python
# app/services/auth_service.py
@staticmethod
def logout(user_cd: str, token: str = "", repo: AuthRepository | None = None) -> None:
    _repo = repo or AuthRepository()
    if token:
        _repo.revoke_token(token)  # 精确删除当前 token
    else:
        _repo.revoke_user_tokens(user_cd)  # 删除用户所有 token
    _repo.add_access_log(user_cd=user_cd, action="LOGOUT", detail="用户登出")
    db.session.commit()
```

### 4.3 会话验证流程

```
┌─────────────────────────────────────────────────────────────┐
│  GET /api/v1/session                                       │
│  Authorization: Bearer <token>                              │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│  AuthService.get_session(token)                             │
│  1. validate_token(token)  ← JWT 验证（签名、过期）         │
│  2. if not is_token_active(token):  ← 数据库活跃性检查      │
│        return None  ← token 已被删除或过期                   │
│  3. 查询用户信息                                            │
│  4. 返回用户信息                                            │
└────────────────────┬────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────────────┐
│  数据库操作                                                 │
│  SELECT COUNT(*) FROM tmc41_acclog                          │
│  WHERE action='ACTIVE_TOKEN'                                │
│    AND detail = '<token>'                                   │
│    AND startdate > NOW();  ← 检查是否仍活跃                 │
└─────────────────────────────────────────────────────────────┘
```

**关键代码：**
```python
# app/services/auth_service.py
@staticmethod
def get_session(token: str) -> dict[str, Any] | None:
    _repo = repo or AuthRepository()
    
    # 1. JWT 验证
    payload = AuthService.validate_token(token)
    if payload is None:
        return None
    
    # 2. 数据库活跃性检查
    if not _repo.is_token_active(token):
        return None
    
    # 3. 查询用户信息
    user = _repo.get_user(payload["user_code"])
    if user is None:
        return None
    
    return {"user_code": user.user_cd, "user_name": user.user_nm, ...}
```

## 5. 关键技术点

### 5.1 精确 Token 管理

**问题：** 按用户删除所有 token 会导致允许多点登录时后登录设备踢掉前一台。

**解决方案：** 按 JWT token 字符串精确删除。

```python
# app/repositories/auth_repository.py
@staticmethod
def revoke_token(token: str) -> None:
    """撤销指定 token + 清理所有过期 token。"""
    db.session.query(AccLog).filter(
        AccLog.action == "ACTIVE_TOKEN",
        AccLog.detail == token,  # 按 JWT 精确匹配
    ).delete(synchronize_session=False)
    AuthRepository.clean_expired_tokens()
```

### 5.2 过期 Token 自动清理

**问题：** token 过期后不被删除，导致数据库累积无效记录。

**解决方案：** 在登录前和登出时自动清理。

```python
# app/repositories/auth_repository.py
@staticmethod
def clean_expired_tokens() -> None:
    """清理所有已过期的 token 记录。"""
    now = datetime.now(timezone.utc).replace(tzinfo=None)
    db.session.query(AccLog).filter(
        AccLog.action == "ACTIVE_TOKEN",
        AccLog.startdate <= now,  # 删除已过期 token
    ).delete(synchronize_session=False)
```

**触发时机：**
- 每次登录前：`AuthService.login()` 调用 `clean_expired_tokens()`
- 每次登出时：`revoke_token()` 调用 `clean_expired_tokens()`

### 5.3 双重会话验证

**问题：** 登出后 JWT 本身未过期，仍可访问 API。

**解决方案：** JWT 验证 + 数据库活跃性检查。

```python
# app/services/auth_service.py
@staticmethod
def get_session(token: str) -> dict[str, Any] | None:
    payload = validate_token(token)  # 第一层：JWT 验证
    if not is_token_active(token):  # 第二层：数据库活跃性验证
        return None
    # ...
```

**效果：**
- JWT 过期 → 第一层拦截
- Token 被删除 → 第二层拦截
- 确保登出后立即失效

## 6. 设计决策与权衡

### 6.1 复用 acclog 表 vs 新建 token 表

| 方案 | 优势 | 劣势 | 选择 |
|------|------|------|------|
| **复用 acclog** | 无需新建表，最小化变更 | 字段语义不直观，需额外文档说明 | ✅ 选择 |
| **新建 token 表** | 语义清晰，独立管理 | 需要数据库迁移，增加表数量 | ❌ 未选 |

**决策理由：** 最小化数据库变更是重构原则之一，`detail` 和 `startdate` 字段恰好可满足需求。

### 6.2 允许多点登录：保留旧 token vs 删除旧 token

| 策略 | 行为 | 选择 |
|------|------|------|
| **保留旧 token** | 各设备独立持有 token，互不影响 | ✅ 选择 |
| **删除旧 token** | 后登录设备踢掉前一台，变成单点登录 | ❌ 未选 |

**决策理由：** "允许多点登录" 语义要求各设备独立，删除旧 token 会变成单点登录。

### 6.3 禁止多点登录：拒绝新登录 vs 踢旧会话

| 策略 | 行为 | 选择 |
|------|------|------|
| **拒绝新登录** | 检测到有活跃 token，第二台直接拒绝 | ✅ 选择 |
| **踢旧会话** | 检测到有活跃 token，踢掉旧设备允许新设备登录 | ❌ 未选 |

**决策理由：** "禁止多点登录" 语义要求同一时刻只有一台设备在线，踢旧会话语义不符。

### 6.4 前端退出：await vs 不 await

| 策略 | 行为 | 选择 |
|------|------|------|
| **await** | 等待后端退出完成再跳转，确保 token 被删除 | ✅ 选择 |
| **不 await** | 立即跳转，后端请求可能未完成，token 可能残留 | ❌ 未选 |

**决策理由：** 必须确保退出请求完成，否则数据库会有残留 token。

## 8. 安全考虑

### 8.1 Token 安全

- **JWT 签名：** 使用 HS256 算法，密钥存储在环境变量
- **Token 过期：** 默认 8 小时（28800 秒），可通过 `JWT_EXPIRATION_SECONDS` 配置
- **Token 存储：** 前端 localStorage 存储，需防范 XSS 攻击

### 8.2 数据库安全

- **Token 存储：** `detail` 字段存储完整 JWT，需防范 SQL 注入（使用 ORM）
- **过期清理：** 自动清理过期 token，避免长期累积
- **日志审计：** 登录/登出操作写入 `acclog` 表，可追溯

### 8.3 接口安全

- **认证装饰器：** `@login_required` 强制要求所有受保护接口带 token
- **Token 验证：** JWT 验证 + 数据库活跃性检查双重保障
- **参数校验：** 使用 Pydantic 进行请求参数校验

### 9.2 缓存策略

- **系统参数缓存：** `allowmultilogon` 参数可缓存，避免每次登录都查数据库

- **登录前清理：** 每次登录前清理过期 token，确保数据量可控
- **登出时清理：** 每次登出时清理过期 token，及时清理
- **定时清理：** 可考虑定时任务（如每天凌晨）清理过期 token（可选）
