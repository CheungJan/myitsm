> 注：BaseModel（base.py）为公共基类，提供 created_at/updated_at 时间戳，不计入业务模型统计。

### 统一设计策略

1. **BaseModel 时间戳**：所有模型继承 `BaseModel`，自动包含 `created_at`/`updated_at` 字段，对应原 Oracle GENDATE/UPDDATE
2. **字符串宽度统一放宽**：Oracle `CHAR(6)/VARCHAR2(20)` → PostgreSQL `String(20/50)`，避免截断
3. **复合主键改代理键**：多对多关联表使用 `Integer` 自增主键 + `UniqueConstraint`，更适合 ORM
4. **P0 优化字段已全部落地**：`customer_status`（P0-2）、`asset_type/recycle_status`（P0-3）、`CustomerHistory`（P0-4）、`source_type`（P0-1）；优化方案1未落地的 SourceType/VerifiedAt/PreplanId/ValidUntil、优化方案4未落地的 ASSET_STATUS/CREATED_FROM/SOURCE_ID/WARRANTY_EXPIRE、优化方案3未落地的 CHANGE_REASON/DEVICE_CHANGE_ID 均已补全
5. **审计字段统一**：`opercd`（操作员）、`useflg`（有效标志）在所有等价迁移表中保留
6. **Oracle 原表字段恢复**：已将 28 张表的 155 个缺失字段全部恢复，确保数据迁移时字段完整对应
