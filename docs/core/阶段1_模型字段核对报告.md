# 阶段1 模型字段与数据库字典核对报告

## 核对依据

- 模型文件：`app/models/system.py`、`app/models/master.py`、`app/models/itsm.py`
- 数据库字典：`docs/core/数据库字典_精简后_最终版.md`（基于原 Oracle CCGL_MIG Schema）
- 优化方案：`docs/core/ITSM重构项目需求设计文档.md`

## 核对结论

**阶段1模型为架构骨架层，覆盖核心业务字段，非全量字段迁移。** 设计策略：

1. **核心字段齐全**：主键、状态字段、外键关联均正确映射
2. **字符串宽度统一放宽**：Oracle CHAR(6)/VARCHAR2(20) → PostgreSQL String(20/50)，避免截断
3. **复合主键改为代理键**：关联表使用 Integer 自增主键 + UniqueConstraint，更适合 ORM 操作
4. **P0优化字段已加入**：customer_status（P0-2）、asset_type/recycle_status（P0-3）、CustomerHistory（P0-4）、source_type（P0-1）
5. **BaseModel 统一时间戳**：created_at/updated_at 对应原 GENDATE/UPDDATE

## 统一省略的字段类别

以下字段在所有表中统一省略，将在后续阶段按需补充：

| 字段 | 原用途 | 省略原因 |
|------|--------|---------|
| WHTRANSFLG | 仓储数据同步标志 | 仓储模块在阶段3实现 |
| STTRANSFLG | 门店数据同步标志 | 门店同步在阶段3实现 |
| OPERCD | 操作员（审计字段） | BaseModel 的 created_at/updated_at 已覆盖时间审计 |
