## 一、批量数据修复

### 1.1 孤儿设备报废（1,230 台）

**问题**：设备 `sflg='1'`（已使用）但无 `CustPosRl`（客户分配）、无 `tmm44_pos_r_eid`（BOM 配件归属），状态矛盾。

**根因**：PB 生产模块中设备生产完成即置 `sflg='1'`，这批设备从未装机到客户。

**操作**：
```sql
-- Step 1: 插入报废轨迹（先读旧值 sflg='1'，再写 n_sflg='2'）
INSERT INTO tmm43_eid_track (...) SELECT ... FROM tmm43_eid e
WHERE e.sflg = '1' AND e.useflg = '1'
  AND NOT EXISTS (SELECT 1 FROM tmm35_cust_pos_rl r WHERE r.eid = e.eid)
  AND NOT EXISTS (SELECT 1 FROM tmm44_pos_r_eid p WHERE p.eid = e.eid);

-- Step 2: 更新设备主表
UPDATE tmm43_eid SET sflg = '2', whcd = NULL, updated_at = NOW()
WHERE sflg = '1' AND useflg = '1' AND <同上条件>;
```

**结果**：插入 1,189 条轨迹（41 个 EID 有双 itemcd 行，各共享一条），1,230 台设备 `sflg`：1→2。

**验证**：
```sql
SELECT COUNT(*) FROM tmm43_eid e
WHERE e.sflg = '1' AND e.useflg = '1'
  AND NOT EXISTS (SELECT 1 FROM tmm35_cust_pos_rl r WHERE r.eid = e.eid)
  AND NOT EXISTS (SELECT 1 FROM tmm44_pos_r_eid p WHERE p.eid = e.eid);
-- 预期：0
```

**回滚**：
```sql
UPDATE tmm43_eid SET sflg = '1', whcd = NULL, updated_at = NOW()
WHERE sflg = '2' AND eid IN (<原孤儿EID列表>);
DELETE FROM tmm43_eid_track WHERE remark LIKE '%批量报废-历史遗留孤立设备%';
```

### 1.2 门店退回设备状态同步（1,650 台）

**问题**：`CustPosRl.useflg='0'` + `asset_status='RETURNED'`（门店已退回），但 `Eid.sflg` 仍为 `'1'`（已使用），`asset_owner` 仍为 `'03'`（客户）。

**根因**：PB 退回流程只更新 `CustPosRl`，未联动更新 `Eid.sflg` 和 `asset_owner`。

**操作**：
```sql
INSERT INTO tmm43_eid_track (...) SELECT ... FROM tmm43_eid e
WHERE e.sflg = '0' AND e.asset_owner = '01'
  AND EXISTS (SELECT 1 FROM tmm35_cust_pos_rl r
              WHERE r.eid = e.eid AND r.useflg = '0' AND r.asset_status = 'RETURNED')
  AND NOT EXISTS (SELECT 1 FROM tmm43_eid_track t
                  WHERE t.eid = e.eid AND t.remark LIKE '%门店退回设备状态同步%');
```

**结果**：插入 1,402 条轨迹（250 条已有旧 track），1,650 台设备 `sflg`：1→0，`asset_owner`→`01`。

**验证**：
```sql
SELECT COUNT(*) FROM tmm35_cust_pos_rl r
JOIN tmm43_eid e ON e.eid = r.eid
WHERE r.useflg = '0' AND r.asset_status = 'RETURNED' AND e.sflg = '1';
-- 预期：0
```

**回滚**：
```sql
DELETE FROM tmm43_eid_track WHERE remark LIKE '%门店退回设备状态同步%';
-- Eid 主表回滚需从 track 记录的 sflg='1' / asset_owner='03' 还原
```

### 1.3 EID 前导/尾随空格清理

**问题**：部分 EID 含空格，精确匹配 `=` 查不到，模糊 `LIKE` 能查到。

**操作**：
```sql
UPDATE tmm43_eid SET eid = TRIM(eid) WHERE eid != TRIM(eid);          -- 1 条
UPDATE tmm43_eid_track SET eid = TRIM(eid) WHERE eid != TRIM(eid);   -- 2 条
UPDATE tmm44_pos_r_eid SET eid = TRIM(eid) WHERE eid != TRIM(eid);   -- 80 条
```

**验证**：
```sql
SELECT 'tmm43_eid', COUNT(*) FROM tmm43_eid WHERE eid != TRIM(eid)
UNION ALL SELECT 'tmm43_eid_track', COUNT(*) FROM tmm43_eid_track WHERE eid != TRIM(eid)
UNION ALL SELECT 'tmm44_pos_r_eid', COUNT(*) FROM tmm44_pos_r_eid WHERE eid != TRIM(eid);
-- 预期全部 0
```

## 二、代码层自动纠正机制

### 2.1 `plan_refid` 三级取值（system_service.py:369-398）

**触发**：EID 列表页 `/api/v1/eid` 每次查询。

**逻辑**：
1. 优先查 `tit15_maintenance_renovate.new_device_id` → `renew_id`
2. 其次查 `EidTrack` type='C'，条件 `change_date >= eid.gendate`
3. 以上均无 → 空

**影响字段**：`plan_refid`（列表页关联单号列）。

---

### 2.2 C 记录实时纠正（system_service.py:508-530）

**触发**：EID 历史详情 `/api/v1/eid/<itemcd>/<eid>/tracks` 每次请求。

**逻辑**：
- `change_date < gendate` → 从 `CustPosRl.posupddate` 修正日期
- `refid` 与翻新单 `renew_id` 不一致 → 替换为正确单号，标记 `_refid_corrected=True`
- 纠正后按 `(change_date, seqno)` 重新排序

---

### 2.3 BOM 配件归属解析（system_repository.py:626-731）

**触发**：资产台账 `/api/v1/assets` 每次查询。

**逻辑**：
1. 主查询 LEFT JOIN `bom_cust_subq`（仅用于位置筛选和 useflg 过滤）
2. 结果页中无直接 `CustPosRl` 的设备 → 批量查 `tmm44_pos_r_eid` → 父设备 `CustPosRl` → `Customer`
3. 解析到的归属信息仅填充当前页（≤per_page 行），非全表

**影响字段**：`host_eid`、`cust_nm`、`cust_card`（BOM 配件行）。

---

### 2.4 BOM 门店退回状态（system.py:614-630）

**触发**：BOM 配件明细 `/api/v1/assets/bom` 每次请求。

**逻辑**：
- 查整机的 `CustPosRl.useflg='1'` 是否存在
- 存在 → `active` = 配件自身有效且门店有效
- 不存在 → `store_returned=True`，前端显示"门店已退回"
