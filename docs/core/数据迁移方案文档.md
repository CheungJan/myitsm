## 一、迁移策略总览

### 1.1 方案选择

| 方案 | 描述 | 选择 |
|------|------|------|
| 方案A | Oracle 整库转 PostgreSQL，再改造表结构 | ❌ 不采用 |
| **方案B** | **SQLAlchemy 模型直接生成 PostgreSQL 空表，再从 Oracle 导入数据** | **✅ 采用** |

**方案B优势**：
- 表结构基于重构后的优化模型（已包含新增字段、类型优化、代理主键等）
- 无需转换 Oracle 存储过程/序列/触发器（Python Service 层已完全替代）
- 表结构干净一致，避免 Oracle 残留结构的兼容问题

### 1.2 迁移范围

| 项目 | Oracle 旧库 | PostgreSQL 新库 | 说明 |
|------|------------|-----------------|------|
| 业务表 | ~150张（CCGLPDB + LGREPORTPDB） | 124张（SQLAlchemy 模型） | 合并去重 + 优化 |
| 存储过程 | ~50+ | 0 | Python Service 层替代 |
| 序列 | ~30+ | 0 | SQLAlchemy 自增/字符串ID替代 |
| 触发器 | ~20+ | 0 | Python Repository 层替代 |
| 视图 | ~10+ | 按需 | 后续根据报表需求创建 |

### 1.3 不迁移项

- **存储过程**：全部由 Python `app/services/` 替代
- **序列**：SQLAlchemy 自动管理自增主键
- **触发器**：审计字段（`opercd`/`upddate`）由 Python Repository 层统一处理
- **DB Link**：原双库（`@CCGL_23`/`@CCGL_24`）已合并为单库，无需跨库引用
- **临时表/中间表**：仅用于历史报表生成，不迁移

## 三、数据迁移流程

### 3.1 总体流程

```
步骤1: PostgreSQL 空表已生成（已完成）
    ↓
步骤2: Oracle 数据导出（按表分批导出 CSV/JSON）
    ↓
步骤3: 数据清洗转换（字段映射、类型转换、编码处理）
    ↓
步骤4: 数据导入 PostgreSQL（分批插入 + 断点续传）
    ↓
步骤5: 数据校验（行数核对 + 关键字段抽检）
    ↓
步骤6: 联调测试（后端API连接新库验证）
```

### 3.2 Oracle → PostgreSQL 数据类型映射

| Oracle 类型 | PostgreSQL 类型 | SQLAlchemy 类型 | 说明 |
|------------|-----------------|-----------------|------|
| VARCHAR2(n) | VARCHAR(n) | String(n) | 直接映射 |
| NUMBER(p,s) | NUMERIC(p,s) | Numeric(p,s) | 精度保持 |
| NUMBER (无精度) | BIGINT / INTEGER | Integer / BigInteger | 按实际值域选择 |
| DATE | TIMESTAMP | DateTime | Oracle DATE 含时间 |
| CLOB | TEXT | Text | 长文本 |
| BLOB | BYTEA | LargeBinary | 二进制 |
| CHAR(1) | VARCHAR(1) | String(1) | 标志位字段 |
| NVL(x, y) | COALESCE(x, y) | — | SQL 函数映射 |
| SYSDATE | NOW() | datetime.now(UTC) | Python 层处理 |
| TO_DATE() | TO_TIMESTAMP() | — | SQL 函数映射 |
| DECODE() | CASE WHEN | — | SQL 函数映射 |
| ROWNUM | LIMIT/OFFSET | — | 分页语法 |

### 3.3 字段映射特殊处理

| 场景 | Oracle 字段 | PostgreSQL 字段 | 处理方式 |
|------|-----------|-----------------|---------|
| 主键变更 | 复合主键 | 代理主键（自增ID） | 数据导入时自动生成新ID |
| 新增字段 | 不存在 | `source_type`/`submit_time` 等 | 导入时设默认值 |
| 字段放宽 | VARCHAR2(10) | String(50) | 直接导入，无需处理 |
| 时间戳字段 | 不存在 | `created_at`/`updated_at` | 导入时设为当前时间 |
| 密码字段 | 明文 | `password_hash` | 导入时使用 werkzeug 加密 |
| 省略字段 | WHTRANSFLG/EXEPATH 等 | 不存在 | PB 专属字段，不导入 |

## 四、分批迁移策略

### 4.1 按表分级

| 级别 | 数据量预估 | 表示例 | 迁移策略 |
|------|----------|--------|---------|
| 小表 | <1万行 | 字典表（TIT01-06）、配置表（TMC71） | 全量一次导入 |
| 中表 | 1-100万行 | 业务主表（TIT10、TMM22、TWH13） | 分批5000行/批 |
| 大表 | >100万行 | 历史记录、日志表 | 分批5000行 + 可选只迁近N年 |

### 4.2 分批导入模板

```python
"""Oracle → PostgreSQL 数据迁移脚本模板。"""

import csv
from datetime import UTC, datetime

import psycopg2

BATCH_SIZE = 5000
PG_CONN = "dbname=myitsm user=cheungjan host=localhost port=5432"


def migrate_table(
    csv_file: str,
    pg_table: str,
    column_mapping: dict[str, str],
    transform_fn=None,
):
    """通用表迁移函数。

    Args:
        csv_file: Oracle 导出的 CSV 文件路径
        pg_table: PostgreSQL 目标表名
        column_mapping: Oracle列名 → PostgreSQL列名 映射
        transform_fn: 可选的行级数据转换函数
    """
    conn = psycopg2.connect(PG_CONN)
    cur = conn.cursor()

    with open(csv_file, "r", encoding="utf-8") as f:
        reader = csv.DictReader(f)
        batch = []
        total = 0

        for row in reader:
            # 字段映射
            pg_row = {}
            for ora_col, pg_col in column_mapping.items():
                pg_row[pg_col] = row.get(ora_col)

            # 补充时间戳
            now = datetime.now(UTC).isoformat()
            pg_row.setdefault("created_at", now)
            pg_row.setdefault("updated_at", now)

            # 自定义转换
            if transform_fn:
                pg_row = transform_fn(pg_row)

            batch.append(pg_row)

            if len(batch) >= BATCH_SIZE:
                _insert_batch(cur, pg_table, batch)
                conn.commit()
                total += len(batch)
                print(f"  {pg_table}: 已导入 {total} 行")
                batch = []

        # 处理剩余数据
        if batch:
            _insert_batch(cur, pg_table, batch)
            conn.commit()
            total += len(batch)

    print(f"  {pg_table}: 导入完成，共 {total} 行")
    cur.close()
    conn.close()


def _insert_batch(cur, table: str, rows: list[dict]):
    """批量插入。"""
    if not rows:
        return
    columns = list(rows[0].keys())
    placeholders = ", ".join(["%s"] * len(columns))
    col_str = ", ".join(columns)
    sql = f"INSERT INTO {table} ({col_str}) VALUES ({placeholders})"
    values = [tuple(row[c] for c in columns) for row in rows]
    cur.executemany(sql, values)
```

### 4.3 断点续传机制

```python
"""断点续传支持。"""

import json
from pathlib import Path

PROGRESS_FILE = "migration_progress.json"


def load_progress() -> dict:
    if Path(PROGRESS_FILE).exists():
        return json.loads(Path(PROGRESS_FILE).read_text())
    return {}


def save_progress(table: str, last_offset: int, status: str):
    progress = load_progress()
    progress[table] = {"last_offset": last_offset, "status": status}
    Path(PROGRESS_FILE).write_text(json.dumps(progress, indent=2))


def get_start_offset(table: str) -> int:
    progress = load_progress()
    if table in progress and progress[table]["status"] != "completed":
        return progress[table]["last_offset"]
    return 0
```

### 6.3 校验脚本模板

```python
"""迁移后数据校验脚本。"""


def verify_migration(ora_conn, pg_conn, tables: list[str]):
    """逐表校验行数一致性。"""
    results = []
    for table in tables:
        # Oracle 行数
        ora_cur = ora_conn.cursor()
        ora_cur.execute(f"SELECT COUNT(*) FROM {table.upper()}")
        ora_count = ora_cur.fetchone()[0]

        # PostgreSQL 行数
        pg_cur = pg_conn.cursor()
        pg_cur.execute(f"SELECT COUNT(*) FROM {table.lower()}")
        pg_count = pg_cur.fetchone()[0]

        status = "PASS" if ora_count == pg_count else "FAIL"
        results.append({
            "table": table,
            "oracle": ora_count,
            "postgresql": pg_count,
            "status": status,
        })
        print(f"  {table}: Oracle={ora_count}, PG={pg_count} [{status}]")

    failed = [r for r in results if r["status"] == "FAIL"]
    print(f"\n校验完成: {len(results)} 张表, "
          f"通过 {len(results) - len(failed)}, 失败 {len(failed)}")
    return results
```

## 九、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 数据量过大导致内存溢出 | 导入中断 | 分批5000行/批 + 断点续传 |
| 字符编码不一致 | 乱码 | 统一使用 UTF-8，导出时指定编码 |
| Oracle DATE 含时间信息 | 时间丢失 | 使用 TIMESTAMP 类型接收 |
| 外键约束导致导入顺序错误 | 导入失败 | 按依赖顺序分批导入（第五章） |
| 主键冲突（代理主键替换） | 导入失败 | 使用序列自增，不导入旧复合主键 |
| Oracle 特殊字符/空值处理 | 数据不一致 | 导入前清洗（空字符串→NULL） |
